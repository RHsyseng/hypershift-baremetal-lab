= Infraenv Deployment
include::_attributes.adoc[]
:profile: telco-hypershift-baremetal-lab

In this section, we prepare an Infraenv environment as a prerequisite to leverage the agent provider.

This compoment will allow the assisted service to generate an ISO we'll mount on the nodes to boot them and drive their installation.

In the following YAML, we will replace the variables PULL_SECRET and SSH_PUB_KEY with their corresponding values

[#create-infraenv]
== Create Infraenv

We will render a YAML such as the following:

[.console-input]
[source,bash,subs="attributes+,+macros"]
-----
apiVersion: v1
kind: Namespace
metadata:
  name: clusters-${CLUSTER}
---
apiVersion: v1
data:
 .dockerconfigjson: ${PULL_SECRET}
kind: Secret
metadata:
  name: ${CLUSTER}-pull-secret
  namespace: clusters-${CLUSTER}
type: kubernetes.io/dockerconfigjson
---
apiVersion: v1
kind: Secret
metadata:
  name: ${CLUSTER}-ssh-key
  namespace: clusters-${CLUSTER}
stringData:
  id_rsa.pub: ${SSH_PUB_KEY}
---
---
apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata:
  creationTimestamp: null
  name: capi-provider-role
  namespace: clusters-${CLUSTER}
rules:
- apiGroups:
  - agent-install.openshift.io
  resources:
  - agents
  verbs:
  - '*'
---
apiVersion: agent-install.openshift.io/v1beta1
kind: InfraEnv
metadata:
  name: ${CLUSTER}
  namespace: clusters-${CLUSTER}
spec:
  pullSecretRef:
    name: ${CLUSTER}-pull-secret
  sshAuthorizedKey: ${SSH_PUB_KEY}
  agentLabels:
    cluster-name: clusters-${CLUSTER}
  nmStateConfigLabelSelector:
    matchLabels:
      cluster-name: clusters-${CLUSTER}
-----

[.console-input]
[source,bash,subs="attributes+,+macros"]
-----
export CLUSTER=myhypershift
export PULL_SECRET=$(cat ~/openshift_pull.json | tr -d [:space:]| base64 -w0)
export SSH_PUB_KEY=$(cat ~/.ssh/id_rsa.pub)
envsubst < scripts/INFRAENV/infraenv.yaml.sample | oc create -f -
-----

Expected Output

```
namespace/clusters-myhypershift created
secret/myhypershift-pull-secret created
secret/myhypershift-ssh-key created
role.rbac.authorization.k8s.io/capi-provider-role created
infraenv.agent-install.openshift.io/myhypershift created
```

After a moment, we can check the resulting object, and in particular see that in `status/isoDownloadURL` a URL for downloading an ISO was generated. This is the iso that can be mounted on the bare metal nodes.

[.console-input]
[source,bash,subs="attributes+,+macros"]
-----
export CLUSTER=myhypershift
oc get infraenv -n clusters-$CLUSTER $CLUSTER -o yaml
-----
